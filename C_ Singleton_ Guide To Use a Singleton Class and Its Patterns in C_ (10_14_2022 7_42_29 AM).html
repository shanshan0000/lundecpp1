    What Is a Singleton in C++?

    As the name “singleton” suggests, the meaning of this word is defining a class only one time in the complete program which means it cannot have multiple classes. It allows the other classes to access themselves, as it provides the global access point. Hence, we can conclude that in C++ singleton class is used when you have a single type of information for all the users.

    Singleton C++: Conditions of Implementing a Singleton Class in C++

    There are some conditions without which a singleton class cannot be implemented in C++. Below we bring you the two main conditions which are most important to use a singleton class.

    • single instance.
    • global access point.

    – Single Instance

    The most common and important part of the singleton class is that it should provide the instance within the class.  It does not have any other class also outer classes and sub-classes should be prevented to create any instance. In C++, this can be done by making the default constructor private, this will restrict the other objects to use the new operator.

    – Global Access Point

    In C++, the instance of a singleton class should provide global access to other classes so that each class can access the singleton class easily. This is only possible if we make a static method and create an access specifier public.

    – Instances Syntax:

    //A singleton class should have public visibility
    //so that complete application can use
    public class (class name) {
    //static instance of class globally accessible
    public static (class name) instance = new (class name) ();
    private (class name) () {
    // private constructor so that class
    //cannot be instantiated from outside
    //this class
    }
    }

    C++ Singleton: Ways to Initialize the Singleton Class

    There are two ways to initialize a singleton class.

    • Early initialization
    • Lazy initialization

    – Early/Eager Initialization:

    Early initialization also known as eager or non-lazy initialization is the simplest method to initialize the class. In this method, we initialize a class whether it is used or not at the time of startup unlike keeping it stored in memory for future use at user demand. Although it is the simplest method it has the limitation that the class will be initialized whether it is used or not which will use more space in memory and time.

    Early/Eager Initialization Code Line:

    private static readonly Singleton singleInstance = new Singleton();

    – Lazy Initialization:

    It is the most preferred type of initialization in a singleton class in which the class is initialized only when it is required. This type of initialization saves memory and time. It is the concept that is commonly used to initialize the object at the point when it is needed. Lazy initialization is applied when the object is very rarely used.

    Lazy Initialization Code line

    private static readonly Lazy<Singleton> Instancelock = new Lazy<Singleton>(() => new Singleton());

    C++ Singleton Methods To Create a Singleton Design Pattern

    In C++, a singleton design pattern can be created by different techniques. You might be thinking about what those techniques are. After thoroughly asking our programming experts, we brought you the easiest explanations of four different techniques to create a singleton design pattern C++. Each of the methods has some limitations which we covered by suggesting the use of other methods.

    Enough chatting; let’s show you the methods:

    • Classic implementation using a private constructor.
    • By making get-instant () synchronized
    • Eager instantiation method
    • Double checked locking method

    – Classic Implementation Using a Private Constructor:

    The main thing in this type of method for creating singleton design patterns is that you have to make the constructor private in your program. This will help us to restrict the other classes to create an object.

    Classic Implementation Using a Private Constructor: Sample Program

    Have a look at the sample program we bring to you for better understandings:

    Classic Implementation Using a Private Constructor: Code
    // creating a singleton design pattern by using a private constructor
    class Singleton
    {
    private static Singleton obj;
    // private constructor to force use of getInstance() to create Singleton object
    private Singleton() {}
    public static Singleton getInstance()
    {
    if (obj==null)
    obj = new Singleton();
    return obj;
    }
    }

    Classic Implementation Using a Private Constructor: Explanation

    In the above code, the concept of lazy initialization is applied. Here, get instance() static is declared. When we call this function for the very first time the singleton object will be created.

    After the first call, whenever we call this function it will return the same object that we have created in the first call.

    Classic Implementation Using a Private Constructor: Limitations

    Although it is the simplest way to create a singleton pattern C++, it also has some limitations. This method of creating a singleton design pattern is not thread-safe which will lead to creating multiple objects.

    – By Making Get-instant () Synchronized

    To overcome the limitations of the above method, we use this method of synchronization. In this method, we use the private constructor and synchronization techniques to make it thread-safe.

    Making Get-Instant() Synchronized: Sample Program

    Have a look at the sample program we bring to you for better understandings:

    Code
    // Thread Synchronization in singleton design pattern
    class Singleton
    {
    private static Singleton obj;
    private Singleton() {}
    public static synchronized Singleton getInstance()
    {
    if (obj==null)
    obj= new singleton();
    return obj;
    }
    }

    Making Get-Instant() Synchronized: Explanation

    This method will make sure that the program is not executing multi-thread at a time. It ensures that get Instance () is using one thread at a time. The code for this type of method is the same as above, it just added one keyword which is synchronized when creating the get instance() function.

    Making Get-Instant() Synchronized: Limitations

    No doubt, this method completely overcomes the above limitation but it also leads to the other limitations. Using synchronization each time while creating an object will use more memory and time, leading the performance of the program to reduce.

    – Eager Instantiation Method

    The main idea of this type of method is the concept of early initialization that is we create objects before we need them and when the need arises we return them. By doing this, we skip the steps of creating the objects to prevent errors during runtime.

    Eager Instantiation Method: Sample Program

    Have a look at the sample program to get an idea of creating a singleton design pattern in C++ using the early instantiation method:

    Code:
    // Static initializer based implementation of singleton design pattern
    class Singleton
    {
    private static Singleton obj = new Singleton();
    private Singleton() {}
    public static Singleton getInstance()
    {
    return obj;
    }
    }

    Eager Instantiation Method: Explanation

    In the code that we just gave you, the concept of early initialization is applied. Here we have skipped the steps of creating the objects to avoid the above limitations of both methods.

    Eager Instantiation Method: Limitations

    The main drawback of this method is that we have to create an object even we do not need it. This can lead to an error for the problems which do not require object creation.

    – Double Checked Locking Method

    The main idea of this type of method is that we will use the lock once on getting instance() when the object is null. By doing this, we can prevent synchronization while creating an object.

    Double Checked Locking Method: Sample Program

    Have a look at the sample program to learn the use of the double-checked method for creating a singleton design pattern in C++:

    Code:
    // Double Checked Locking method for creating a singleton design pattern
    class Singleton
    {
    private volatile static Singleton obj;
    private Singleton() {}
    public static Singleton getInstance()
    {
    if (obj == null)
    {
    // To make thread safe
    synchronized (Singleton.class)
    {
    // check again as multiple threads
    // can reach above step
    if (obj==null)
    obj = new Singleton();
    }
    }
    return obj;
    }
    }

    Eager Instantiation Method: Explanation

    In the code presented above, to reduce the overhead of calling the synchronized method every time, we used the volatile keyword for the object.

    Which Method Is Best for Creating the Singleton Class Design Pattern?

    Among all four methods, the best and most preferred method is creating a singleton design pattern using the double-checked locking method. This method overcomes all the drawbacks of other methods.

    How To Implement the Singleton Class Design Pattern in C++?

    We have already seen the methods to create a singleton class design pattern. Now, we will teach you how we use the code of a singleton class design pattern in the C++ program. In C++ to create a  singleton class design pattern the program is divided into the following three parts:

    • Pre-code
    • Singleton class design pattern code
    • Post-code

    We have already discussed the singleton class design pattern code. So now we will discuss the other two parts of the program to implement a singleton class design pattern in c++.

    – C++ Singleton: Pre-Code

    In C++, to create a singleton class design pattern we write this code just before the singleton class design pattern code.

    Pre-Code Sample Code

    Take a look at the following sample program for better understandings:

    class GlobalClass
    {
    int m_value;
    public:
    GlobalClass(int v = 0)
    {
    m_value = v;
    }
    int get_value()
    {
    return m_value;
    }
    void set_value(int v)
    {
    m_value = v;
    }
    };
    // Default initialization
    GlobalClass *global_ptr = 0;
    void foo(void)
    {
    // Initialization on first use
    if (!global_ptr)
    global_ptr = new GlobalClass;
    global_ptr->set_value(1);
    cout << “foo: global_ptr is ” << global_ptr->get_value() << ‘n’;
    }
    void bar(void)
    {
    if (!global_ptr)
    global_ptr = new GlobalClass;
    global_ptr->set_value(2);
    cout << “bar: global_ptr is ” << global_ptr->get_value() << ‘n’;
    }
    int main()
    {
    if (!global_ptr)
    global_ptr = new GlobalClass;
    cout << “main: global_ptr is ” << global_ptr->get_value() << ‘n’;
    foo();
    bar();
    }

    Pre-Code Example: Explanation

    Here we initialized the global variable which is initialized by default and is re-initialized on its first occurrence.

    – Post-Code

    Post-code comes right after the singleton class design pattern code. It restricts the class for its global pointer. Only the public accessor method is used by the user.

    Post-Code Sample Code

    Check the code written below as an example for the post-code part:

    class GlobalClass
    {
    int m_value;
    static GlobalClass *s_instance;
    GlobalClass(int v = 0)
    {
    m_value = v;
    }
    public:
    int get_value()
    {
    return m_value;
    }
    void set_value(int v)
    {
    m_value = v;
    }
    static GlobalClass *instance()
    {
    if (!s_instance)
    s_instance = new GlobalClass;
    return s_instance;
    }
    };
    // Allocating and initializing GlobalClass’s
    // static data member.  The pointer is being
    // allocated – not the object itself.
    GlobalClass *GlobalClass::s_instance = 0;
    void foo(void)
    {
    GlobalClass::instance()->set_value(1);
    cout << “foo: global_ptr is ” << GlobalClass::instance()->get_value() << ‘n’;
    }
    void bar(void)
    {
    GlobalClass::instance()->set_value(2);
    cout << “bar: global_ptr is ” << GlobalClass::instance()->get_value() << ‘n’;
    }
    int main()
    {
    cout << “main: global_ptr is ” << GlobalClass::instance()->get_value() << ‘n’;
    foo();
    bar();
    }

    Post-Code Example: Explanation

    The above code simply uses a private static pointer and a public accessor method.
